Versioning

This project use triplet version system and worth to mention here. A version compose of specification version, release version and source code version. The distinguishing feature is a version is not outdated by the new version. If an old version is subset of a new version, the new version is called backward compatible.  

Specification version consist API documentation, target platforms and compatibility test suite. Specification version usually have a code name and version number can interchangably use with code name. 

Release version refer to binary distribution version publish to public use. The first release version is 0 and increase by one successively. Release candidate version is -1, beta version is -2 and alpha version is -3.

Source code version are increase by one after significant changes. Source code is irrespective of release version and share across specification and release version.

Module Release Process


A compile distribution is promoted to release status when it pass defined compatibility test for all its target platform.
